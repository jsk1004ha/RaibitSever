RAIBITSERVER · UX SPEC

제품형 SaaS 콘솔 설계 명세

대상은 동아리/학교 팀원이지만 구조는 운영 가능한 developer cloud platform입니다. UI는 배포 상태, 서비스 상태, 리소스 상태를 먼저 말하고 Kubernetes/control-plane 개념은 이벤트와 고급 설정에서만 드러냅니다.

런처

1. IA / Navigation

Global sidebarDashboard, Projects, Deployments, Resources, GitHub, Usage, Audit, Admin, Settings. 모바일에서는 상단 org/project switcher와 하단 수준의 horizontal nav로 축약합니다.
Organization switcher조직명, role, quota 상태를 함께 표시합니다. pending user는 switcher 아래에 “승인 대기 중” banner를 고정합니다.
Project switcher최근 프로젝트, 상태 badge, 검색, “새 프로젝트” CTA. 프로젝트 내부에서는 Overview, Services, Deployments, Resources, Domains, Env, Logs, Audit, Settings tabs.
AdminPending users, User quota, Global usage, Audit logs, Provider health. 일반 member에게는 노출하지 않습니다.
GitHub integrationInstallations, repositories, import, attach, webhook health, PR preview policy.
Usage/quota/audit사용량은 user/org/project 레벨로 분리하고 quota block의 원인과 관리자 요청 CTA를 함께 표시합니다.

Patterns

StatusBadge는 Ready, Building, Queued, Failed, Paused, Provisioning, Attached, Pending, Blocked를 색상+문구+microcopy로 구분합니다. Button은 primary, secondary, ghost, danger. Card/Table/Form/LogViewer는 hairline border, tabular numerics, compact density. Modal은 typed confirmation과 consequence list. Toast는 action result만, long-running 작업은 EventStream으로 이동합니다.

5. 제품화 UX flows

신규 승인 대기가입 → pending dashboard → 허용된 읽기 영역만 표시 → 관리자에게 요청 메시지 → 승인되면 onboarding checklist 재개.
관리자 승인Pending queue → 사용자 맥락 확인 → account type 선택 → quota preset 조정 → approve → audit log 기록 → 사용자 알림.
GitHub import 배포GitHub App 설치 → repo 선택 → Dockerfile scan → service type 확인 → env/resource attach → deploy queued → logs/events → runtime URL.
PR previewPR opened webhook → preview deployment 생성 → preview URL/check plan → PR closed cleanup workflow → cleanup event.
Resource attachcatalog 선택 → plan/size 선택 → provision → masked connection 확인 → service 선택 attach → env diff → redeploy 안내.
Read-only queryconsole 진입 → schema/table 선택 → SELECT 실행 → limit/timeout 결과 → destructive command는 차단 copy.
실패 후 rollbackfailed badge → deployment detail → 로그 severity filter → 이벤트 원인 확인 → rollback modal → previous READY image queue.
Quota 초과CTA disabled → 초과 항목/현재 사용량/필요 quota 표시 → 관리자 요청 → audit/notification.

6. UI copy

Headline
“Repo에서 runtime URL까지, 안전한 기본값으로 배포하세요.”

Empty
“아직 프로젝트가 없습니다. GitHub 저장소를 가져오면 첫 서비스와 preview URL을 함께 만들 수 있습니다.”

Error
“배포가 실패했습니다. Dockerfile build 단계에서 종료되었습니다. 로그에서 실패한 명령을 확인하세요.”

Quota warning
“이번 달 build minutes의 80%를 사용했습니다. 다음 배포는 가능하지만 quota 초과 시 새 배포가 차단됩니다.”

Pending
“관리자 승인 전에는 project 생성, deployment, resource provision이 잠겨 있습니다.”

Secret masking
“보안을 위해 secret 원문은 표시하지 않습니다. 필요한 경우 credential rotation으로 새 값을 발급하세요.”

Confirmation
“rollback club-api를 입력하면 이전 READY image로 되돌립니다. 이 작업은 audit log에 기록됩니다.”

Tooltip
“Preview deployment는 PR이 닫히면 자동 cleanup workflow를 생성합니다.”
